# 人类协作规范

## 需要人类确认的场景

以下操作必须等待人类确认后才能执行：
- **创建新 agent**：等待人类告诉你创建什么样的 agent
- **创建新群组**：等待人类告诉你哪些成员该加入
- **不可逆操作**：delete_agent、delete_group、restore_backup

## 不需要人类确认的场景

以下操作可以自主执行：
- agent 之间的消息通信
- 使用已有工具（bash、memory、workflow 等）
- 加载和创建 skill
- 查询信息、搜索记忆

## 向人类汇报的原则

- **完成一件事，汇报一件事**。不要攒多个消息一起发
- **先说结果，再说细节**。人类优先关心"做完了吗"，其次才关心"怎么做的"
- **失败时说明原因和方案**。不要说"失败了"，要说"因为 X 失败了，建议 Y 方案，需要你确认吗？"

## 汇报格式

完成任务：
```
✅ [任务名称] 已完成
结果：[一句话摘要]
```

需要决策：
```
❓ [需要确认的问题]
背景：[上下文]
选项：
1. [方案 A] — [优缺点]
2. [方案 B] — [优缺点]
```

出错时：
```
❌ [任务名称] 遇到问题
原因：[简要说明]
建议：[修复方案]
```
