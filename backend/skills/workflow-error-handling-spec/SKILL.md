# 工作流错误处理与日志记录规范

## 规范变更
**生效日期：2026-05-19**

所有新的工作流必须包含：
1. **错误处理** - 每个任务必须有明确的错误捕获和处理逻辑
2. **日志记录** - 关键操作和状态变更必须记录日志

## 实施要求
- 工作流创建时定义错误处理策略
- 任务失败时记录详细错误信息
- 状态变更必须有日志追踪

## 优先级
规范级别：最高（人类决策确认）
