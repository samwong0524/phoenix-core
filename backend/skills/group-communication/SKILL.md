# 群组沟通规范

## 核心原则

- **有人类在等 → 优先处理人类请求**，完成后再和其他 agent 同步
- **没有新输入 → 保持沉默**。除非你有充分的理由（工作流任务完成、需要其他 agent 配合），否则不主动发言
- **一条消息只做一件事**。完成一个动作后，发一条确认消息，然后等待

## 发言场景

应该发言：
- 人类在群组中提出了请求
- 工作流中的任务被分配给你，你开始或完成了工作
- 需要其他 agent 提供信息才能继续
- 发现了需要同步的重要状态变化

不应该发言：
- 没有新消息（不要自己接自己的话）
- 纯粹同意或附和（"好的"、"收到"——除非人类或其他 agent 明确要求确认）
- 重复已经说过的内容

## 消息格式

向人类汇报时：
```
已完成 [任务]: [结果摘要]
如需调整或补充请告诉我。
```

向其他 agent 同步时：
```
@[agent_role]: [你需要对方做什么 + 原因]
```

## 群组类型理解

- **P2P 群组**（你和另一个 agent）：直接沟通，可以频繁同步
- **多人群组**（3+ agent + human）：保持消息精简，每条消息都要有信息量
- **含人类的群组**：任何对外可见的进展必须发到含人类的群组
