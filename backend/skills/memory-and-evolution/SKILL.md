# 记忆与自进化

## 记忆工具选择

| 场景 | 工具 |
|------|------|
| 保存重要决策、事实、用户偏好 | `memory_add`（设 importance 4-5） |
| 查找历史上下文 | `memory_search`（优先于猜） |
| 更新过时的信息 | `memory_replace` |
| 删除错误或过时的记录 | `memory_remove` |
| 搜索历史会话 | `session_search` |

## importance 使用指南

- **5（关键）**：不可逆决策、安全策略、用户核心身份信息
- **4（重要）**：架构选择、项目约定、常用工作流
- **3（正常）**：一般有用信息（默认）
- **2（低）**：临时或局部有用
- **1（最低）**：几乎不重要的记录

## 技能创建触发器

在以下情况使用 `create_skill`：
1. **完成非平凡任务后**——提取可复用的工作模式
2. **工具失败 3 次以上**——记录修复方案
3. **发现新平台或工具的使用模式**——保存经验
4. **工作流完成**——总结有效模式

## 技能设计原则

- 名称 kebab-case，清晰描述领域
- 描述一行内说清什么场景下加载此 skill
- 内容聚焦：教 agent 怎么做，而非做什么
- 设置 `autoLoad: true` 仅当该 skill 对大多数 agent 普遍有用
