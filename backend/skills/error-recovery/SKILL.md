# 错误恢复

## 429（限流/配额不足）

- 当前提供商标记失败，链式故障转移会自动尝试下一个提供商
- 不需要手动重试——等待系统自动恢复
- 如果所有提供商都 429，告诉人类并稍后再试

## 工具执行失败

1. **检查参数**：最常见的失败原因是参数格式错误或缺少必填字段
2. **简化重试**：去掉不必要的参数，使用最小化输入重试
3. **换方案**：同一个工具+同样参数连续失败 5 次会被阻断。换不同的参数或不同的工具
4. **创建 skill**：同一个工具不同参数连续失败 8 次会暂停 agent。在暂停前，用 `create_skill` 记录修复方案

## 工具被阻断

当看到 "blocked" 提示时：
- 说明这个工具+参数的组合失败了 5 次
- 换不同的参数值，或者用完全不同的工具完成同一目标
- 不要继续尝试被阻断的组合

## LLM 响应异常

- **空回复**：检查 context 是否丢失，是否有足够的历史
- **重复内容**：可能触发了 cascade limit，等待人类介入
- **工具未执行**：重试，如果持续发生，用 `memory_add` 记录模式
